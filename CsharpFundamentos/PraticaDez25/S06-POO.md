**Programação Orientada a Objetos (POO)**
**Programação Orientada a Objetos (POO):** Um paradigma de programação que utiliza "objetos" para representar dados e comportamentos, promovendo a modularidade, reutilização de código e organização do software.
**Objeto:** Instância de uma classe que representa uma entidade específica com estado e comportamento.
**Classe:** Estrutura que define um tipo de dado personalizado, encapsulando propriedades e métodos relacionados.

**Pilares da POO:**
**Herança:** Mecanismo que permite a criação de uma nova classe (classe derivada) baseada em uma classe existente (classe base), herdando seus membros (propriedades, métodos) e podendo adicionar novos membros ou modificar os existentes.
**Encapsulamento:** Princípio que restringe o acesso direto aos membros de uma classe, permitindo o controle sobre como os dados são acessados e modificados, geralmente através de propriedades e métodos públicos.
**Polimorfismo:** Capacidade de um objeto assumir diferentes formas, permitindo que métodos com o mesmo nome se comportem de maneira diferente dependendo do objeto que os invoca.
**Abstração:** Processo de simplificação de um sistema complexo, focando nos aspectos essenciais e ignorando os detalhes irrelevantes, geralmente através da criação de classes e interfaces que representam conceitos do mundo real.

**Interface:** Contrato que define um conjunto de métodos e propriedades que uma classe deve implementar, sem fornecer a implementação desses membros.
**Composição:** Técnica de design onde uma classe é composta por outras classes, permitindo a criação de objetos complexos a partir de objetos mais simples.
**Agregação:** Tipo especial de associação onde uma classe contém referências a objetos de outras classes, mas esses objetos podem existir independentemente da classe que os agrega.
**Composição vs Agregação:** Composição implica uma relação de "parte-todo" onde a parte não pode existir sem o todo, enquanto agregação permite que as partes existam independentemente do todo.
**Composição vs Herança:** Composição é um princípio onde uma classe é composta por outras classes, promovendo reutilização de código e flexibilidade, enquanto herança é um mecanismo onde uma classe deriva de outra, herdando seus membros.
**Downcasting vs Upcasting:** Upcasting é a conversão de um objeto de uma classe derivada para uma classe base, enquanto downcasting é a conversão de um objeto de uma classe base para uma classe derivada.
**Interface vs Classe Abstrata:** Interfaces definem um contrato que as classes devem seguir, sem fornecer implementação, enquanto classes abstratas podem fornecer implementação parcial e definir membros que devem ser implementados pelas classes derivadas.
**Virtual vs Override:** Virtual é uma palavra-chave usada para declarar um método em uma classe base que pode ser sobrescrito em classes derivadas, enquanto Override é usada para indicar que um método em uma classe derivada está sobrescrevendo um método virtual da classe base.
