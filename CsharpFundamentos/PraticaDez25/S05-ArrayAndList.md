**Array (vetor):** Uma estrutura de dados que armazena uma coleção fixa de elementos do mesmo tipo, acessados por índices numéricos. Métodos comuns incluem: Length, GetLength, Rank, GetValue, SetValue, IndexOf, Sort, Reverse, Copy, BynarySearch.
**List:** Uma coleção dinâmica que pode crescer e encolher conforme necessário, permitindo armazenar elementos do mesmo tipo e oferecendo métodos para manipulação dos dados, como adição, remoção e busca. Métodos comuns incluem: Add, Remove, Insert, Clear, Contains, IndexOf, Sort, Reverse, ToArray, OrderBy, FirstOrDefault, Any, Where, ToList, Find, FindAll, FindLast, FindIndex, FindLastIndex.
**ArrayList:** Uma coleção não genérica que pode armazenar elementos de qualquer tipo, mas não oferece segurança de tipo em tempo de compilação, podendo levar a erros em tempo de execução.
**Array vs List:** Arrays têm tamanho fixo e são mais eficientes em termos de desempenho, enquanto Lists são dinâmicas e oferecem mais flexibilidade e funcionalidades para manipulação de dados.
**ArrayList vs List:** ArrayList é uma coleção não genérica que pode armazenar elementos de qualquer tipo, enquanto List é uma coleção genérica que oferece segurança de tipo em tempo de compilação e melhor desempenho.
**Matriz multidimensional:** Um array que pode ter mais de uma dimensão, permitindo armazenar dados em uma estrutura tabular ou em várias camadas. Exemplos incluem: arrays bidimensionais (2D) e tridimensionais (3D).
**Jagged Array:** Um array de arrays, onde cada elemento do array principal pode ser um array de tamanho diferente, permitindo estruturas de dados irregulares.
**Coleção genérica:** Uma coleção que pode armazenar elementos de qualquer tipo especificado, proporcionando segurança de tipo em tempo de compilação e melhor desempenho. Exemplos incluem: List<T>, Dictionary<TKey, TValue>, HashSet<T>, Queue<T>, Stack<T>, LinkedList<T>, SortedList<TKey, TValue>, SortedDictionary<TKey, TValue>.
**Coleção não genérica:** Uma coleção que pode armazenar elementos de qualquer tipo, mas não oferece segurança de tipo em tempo de compilação, podendo levar a erros em tempo de execução. Exemplos incluem: ArrayList, Stack, SortedList, Queue, Hashtable.
**IEnumerable vs IEnumerator:** IEnumerable é uma interface que permite a iteração sobre uma coleção, enquanto IEnumerator é uma interface que fornece a funcionalidade para percorrer os elementos da coleção.
**List e IEnumerable:** List implementa a interface IEnumerable, permitindo que seus elementos sejam iterados usando loops foreach e outras construções de iteração. O IEnumerable fornece a funcionalidade básica para percorrer coleções de forma genérica.
**Deferred Execution:** Um conceito em LINQ onde a execução de uma consulta é adiada até que os resultados sejam realmente necessários, permitindo otimizações e evitando operações desnecessárias. Exemplos incluem: consultas LINQ que retornam IEnumerable ou IQueryable.
**Boxing vs Unboxing:** Boxing é o processo de converter um tipo de valor em um tipo de referência, enquanto unboxing é o processo inverso, convertendo um tipo de referência de volta para um tipo de valor.
**Capacidade vs Contagem:** Capacidade refere-se ao número total de elementos que uma coleção pode armazenar sem redimensionamento, enquanto contagem é o número atual de elementos presentes na coleção.
**Fila (Queue):** Uma coleção que segue a ordem FIFO (First In, First Out), onde o primeiro elemento adicionado é o primeiro a ser removido.
**Pilha (Stack):** Uma coleção que segue a ordem LIFO (Last In, First Out), onde o último elemento adicionado é o primeiro a ser removido.
**Dicionário (Dictionary):** Uma coleção de pares chave-valor, onde cada chave é única e é usada para acessar o valor correspondente.
**HashSet:** Uma coleção que armazena elementos únicos, sem ordem específica, e é otimizada para operações de busca e inserção rápidas.
**SortedList vs SortedDictionary:** SortedList é uma coleção que mantém os elementos ordenados por chave e permite acesso por índice, enquanto SortedDictionary é uma coleção que também mantém os elementos ordenados por chave, mas não permite acesso por índice.
**LINQ (Language Integrated Query):** Uma funcionalidade do C# que permite consultar coleções de dados de forma declarativa, usando uma sintaxe semelhante a SQL.
**Predicate:** Um delegado que representa um método que retorna um valor booleano, usado para definir condições em consultas LINQ e outros contextos.
**Delegate:** Um tipo que representa referências a métodos com uma assinatura específica, permitindo a passagem de métodos como parâmetros. Exemplos incluem: Func, Action, Predicate. É similar a um ponteiro de função em outras linguagens.
**Lambda Expression:** Uma forma concisa de representar métodos anônimos usando uma sintaxe simplificada, frequentemente usada em consultas LINQ e para definir delegados.
**Índice:** A posição numérica de um elemento dentro de uma coleção, usado para acessar ou modificar o elemento.
**Indexador:** Uma propriedade especial que permite acessar elementos de uma coleção usando uma sintaxe semelhante a arrays, facilitando a manipulação dos dados. Utilizado em classes personalizadas para fornecer acesso baseado em índice. Exemplos incluem: this[int index].
**Params:** Uma palavra-chave que permite passar um número variável de argumentos para um método, facilitando a criação de métodos flexíveis.

**Coleções imutáveis:** Coleções que não podem ser modificadas após sua criação, garantindo que os dados permaneçam constantes e protegidos contra alterações acidentais. Exemplos incluem: ImmutableList, ImmutableDictionary, ImmutableHashSet.
**Span vs Memory:** Span é uma estrutura de dados que representa uma fatia contínua de memória e é usada para manipulação eficiente de dados em tempo de execução, enquanto Memory é uma estrutura que representa uma região de memória que pode ser usada para armazenar dados de forma segura e eficiente, mesmo em contextos assíncronos.
